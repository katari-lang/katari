# MCP static tool binding: `mcp.call` + `katari mcp pull` (v0.1.0, scrap-and-build)

ゴール: **MCP サーバーのツールを、実行時の動的 mint(`mcp.provide`)ではなく、コンパイル時に型付きの
Katari モジュールとして固定できること**。

```
katari mcp pull --url https://server/mcp --out src/github.ktr [--header k=v]... [--oauth [--scope s]]
```

生成物は自己完結の `.ktr` モジュール 1 個: サーバーのツール 1 つにつき**トップレベル**の型付き agent が
1 つ、それらへ接続(`url` + `auth`)を供給する `connect` provider が 1 つ、そして接続の credentials を
ambient に配る `credentials` request が 1 つ。トップレベル agent は何もクローズできないので、旧
`with_tools`(継続にクロージャの record を渡す形)は**削除**し、パッケージの provider idiom
(provider agent + capability request + トップレベル tool agent、`tavily` / `e2b` と同型)に揃えた。
再生成はファイルを上書きする(生成物は成果物であり、手編集しない)。

```katari
import github   // `--out src/github.ktr` の生成モジュール

agent main() -> string {
  // 接続を開く。bare `use`(binder 無し)なので継続の値型は null。K3013 は binder-less `use` に噛まない。
  use github.connect(auth = mcp.oauth(name = "github"))
  // ツールはトップレベル agent、そのまま直接呼ぶ。
  let issue = github.get_issue(owner = "katari-lang", repo = "katari", number = 1)
  issue.title
}
```

生成バインディングは `provide` スコープの**中で**暮らす: `connect` は `mcp.provide` をラップして
スコープを開き、`use handler` で `credentials` を供給し、継続に `null` を渡す。各トップレベル tool は
`credentials()` で接続の `auth` を読み、`mcp.call` を呼ぶ(何もクローズしない)。tool の行と `connect`
の継続の行に `mcp.scope["<literal>"]` が乗り、`connect` 自身の結果がそのスコープを discharge するので、
tool は `connect` の外では呼べない(設計は [docs/2026-07-11-mcp-provide.md](2026-07-11-mcp-provide.md))。

`connect` の継続の行は**混在綴り** `{...(E | mcp.scope["<url>"]), credentials}`: phantom marker の
`scope` は**union 側**(2 つの生成モジュールを 1 ブロックで nest したとき、URL ごとの scope が
arg-union で `scope["a" | "b"]` にマージするのを許す — pin されると 2 つ目で弾かれる)、handle される
`credentials` は**overwrite 側**(handler が discharge するには共有 `E` から pin して外す必要がある。
純 union `| credentials` だと `E` に残り、discharge が K3001 で失敗する — 実測)。2 つのサーバーの
`credentials` request は**モジュールで名前空間化**されるので衝突しない。

> 留意(将来最適化): `connect` は現状スコープを開くために `mcp.provide` の listing 往復を走らせるが、
> 静的バインディングは mint された toolbox を使わない(`mcp.call` 経由)。listing 不要の
> `mcp.open` primitive があればこの一度きりのオーバーヘッドを消せる(未実装 — ここに記録するだけ)。

## 3 層構成

### 1. stdlib prim `mcp.call`(`prelude/mcp.ktr` + runtime)

minted-tool 経路の静的な対応物 — スキーマを見ない直接呼び出し:

```katari
external agent call[literal URL, T](url: URL, auth: auth, tool: string, arguments: json.json) -> T with scope[URL] | prelude.throw[server_error | auth_error | json.decode_error] from "mcp"
```

`T` は `json.decode[T]` / `json.parse_as[T]` と同じくスキーマ指向で、結果にしか現れないので推論できず
**明示 instantiate が必須**(K3016)。literal `URL` generic は引数から推論されるが、明示 `[...]` の
arity には数えられる(混在した「一部だけ明示」は書けない — K3009)ので、呼び出し側は両方書く:
`mcp.call[url_literal, T](url = url_literal, ...)`。生成コードは URL literal をリテラルとして持っている
ので、instantiation と `url` 引数の両方に同じリテラルが出る。

- compiled external は context を持たないので、すべて(`{url, auth, tool, arguments}`)が呼び出しの
  **引数**に乗る。runtime 側は `mcp-reactor` の payload union に `directCall` 変種を 1 つ足しただけで、
  `openPayload` で `prelude.mcp.call` キーから一度だけ判別する(既存 2 キーと同型)。
- スコープ gating は **型レベル**(行の `scope[URL]`、literal URL generic 経由 — literal な `url` なら
  URL ごと、dynamic なら `scope[string]`)。ランタイム規則は「記述子(url + auth)について**生きた
  `provide` スコープを要求する**」で、生きた provide の無い呼び出しは欠けている provide を名指しする
  typed `server_error` で拒否する。生成バインディングはこれを満たすよう `connect` の `provide`
  スコープの中でだけ呼ぶ(§3、[docs/2026-07-11-mcp-provide.md](2026-07-11-mcp-provide.md))。
- `arguments` は `json` **ツリー**。dispatch 時に `jsonValueToJson`(`json.stringify` の裏にある
  総変換; blob-backed な文字列 leaf は store から実体化)で **literal な Json 文書**に下ろし、
  transport には既存の `callTool` オペレーションそのままで渡す(接続キャッシュ・auth ディスパッチ・
  typed throw は全部共用)。
- 返りは runtime 側で **`T` に対してデコード**する — `json.decode[T]` と同じ機構
  (`conformValue` + wire 復元 + typed `decode_error`)を再利用する。`[URL, T]` の instantiation は
  external agent 自身の ambient として engine が reactor へ転送するので、`generics.T` の schema が
  directCall の payload に届く。conform-tree-first:
  - typed な `T`(マップされた output type): `structuredContent` を `T` の **wire form** として復元
    (`jsonToValue`)— `$ref` オブジェクトは**本物の `file` ハンドル**に、record/array/scalar は値に
    なる。plain text は string 形の `T` にだけデコードされる。`T` に conform しなければ typed
    `json.decode_error`。
  - `T = json.json`(outputSchema がマップされない時の codegen の選択): 生 Json を
    `jsonValueFromJson` で **literal にツリーへ持ち上げる**(従来どおり)。`$ref` はツリー内の inert な
    オブジェクトのまま残り、後で `json.decode` を `file` 入りの型で当てると handle に復元できる。
    (json 木は必ず `json.json` の schema に conform するので、conform-tree-first がそのままツリーを返す。)
- conform しない返りは reactor の `complete` override で typed `json.decode_error` throw に整形する
  (値を組む decodeAck seam は settle-time に走り throw できないので、settle 前に判定する)。
- 所有権: typed デコードは**本物の ref** を作るので、resource の release/reown walk が値到達可能性で
  寿命を縛る。run-instance への adoption backstop は `json.json` の inert `$ref`(文字列のまま)だけの
  ための狭い保険に縮む。
- エラーは従来の typed `server_error` / `auth_error` に加え、返りが `T` に conform しない場合の
  `json.decode_error`(行が宣言; ツリー下ろしの失敗も `server_error` 経路のまま)。

### 2. helper subcommand `katari-mcp list-tools`(typescript/mcp)

```
katari-mcp list-tools --url <server> [--header k=v]... [--oauth [--scope s]]
```

SDK client で接続(streamable HTTP → SSE fallback、runtime transport の `clientFor` と同順)し、
`{"tools":[{"name","description","inputSchema","outputSchema"?}]}` を stdout に印字する。
`--oauth` は login と**同じ** interactive flow(`performLogin` — 共有ライブラリ関数)を回すが、
credential は **in-memory のみ**(何も保存しない; pull は dev-time の読み取りなので)。

### 3. CLI `katari mcp pull` + codegen(haskell/cli)

- `Katari.Cli.Command.Mcp` に `pull` verb。helper は login と同じ解決順
  (`KATARI_MCP_BIN` → `node_modules/.bin` walk-up → PATH)で spawn、stderr は inherit
  (authorization URL が見える)、stdout は pipe(JSON)。listing は
  `Katari.Data.JSONSchema` の `FromJSON` でデコード — コンパイラがモデル化する JSON Schema 部分集合が
  そのまま「マップできる範囲」の唯一の権威。
- `Katari.Cli.McpCodegen` が listing → モジュールテキスト。`pull` は runtime 接続を要求しない
  (ローカルにファイルを書くだけ; `login` だけが env API を使う)。

## 型マッピング(実装どおり)

| JSON Schema | Katari |
| --- | --- |
| `string` / `integer` / `number` / `boolean` / `null` | そのまま |
| `array` + `items` | `array[T]` |
| `object` + `properties` | object 型(non-required は `?` フィールド) |
| `object` + schema 値の `additionalProperties` のみ | `record[T]` |
| `anyOf`(全メンバーがマップ可) | union `A \| B` |
| それ以外(`enum` / `const` — literal 型はあるが値マッピングは依然 fallback —、`allOf` / `oneOf`、tuple、空スキーマ、識別子にならないフィールド名) | `json.json` fallback |

注意: デコーダ(`JSONSchema` の `FromJSON`)は `type` 付き `enum` を base type に落とす
(`{"type":"string","enum":[...]}` → `string`)。enum 性は失われるがサーバー側検証は残る。
bare `enum` はモデル外 → 空スキーマ → fallback。

## wire-form 非対称(all-or-nothing の理由)

**パラメータ側**の非対称: `json.encode` は値を **wire form** で埋め込む。つまり型の中に `json.json` が
1 箇所でもあると、そこは `$constructor` タグ付きツリーとして埋まり、サーバーが期待する生のフラグメント
にならない。だからパラメータのマッピングは all-or-nothing:

- **(a) パラメータ**: 完全にマップされたパラメータは `json.encode(value = p)` で埋め込む。
  fallback パラメータは既にツリーなので **as-is** で arguments ツリーに挿す(encode しない)。

**出力側**にはこの非対称は無い(デコードが runtime に移った): ラッパーは常に末尾式
`mcp.call[<literal url>, T](...)` を呼ぶ(`json.decode` の行は消える)。

- **(b) 出力**: `outputSchema` 全体が fallback ゼロでマップされたら `T` = その `<tool>_output` シノニム型
  (runtime が `T` に対してデコード)、それ以外は `T` = `json.json`(生ツリー)。`mcp.call` の行は常に
  `json.decode_error` を持つので、**両方**のラッパー effect 行が `json.decode_error` を含む
  (`json.json` の場合は実際には投げられない — over-approximation)。

codegen テスト(`McpCodegenSpec`)は (a)(b) の違反をそのまま検出する形で書いてある
(fallback パラメータに `json.encode` が現れないこと、typed 出力が `mcp.call[..., <tool>_output]` を
呼ぶこと、部分マップの出力に synonym が生えず `mcp.call[..., json.json]` になること)。

## 生成 body の一様な構築形

arguments オブジェクトは **`record.set` によるフィールド単位の畳み込み 1 本**
(`arguments_0 = record.empty()` → `arguments_n`)。optional パラメータは
`p: T | null ?= null` で受け、`match` の `null` 腕が**キーごと省略**する — 省略と JSON null は
混ざらない(JSON null は `json.json_null()` で別物)。properties を持たない入力スキーマは、
閉じた空 object なら 0 引数、それ以外は `arguments: json.json` のパススルー 1 引数に退化する
(呼べないツールを作らない)。

## 名前マングリング

server 名 → Katari 識別子は決定的: snake_case(camelCase は `_` 挿入で lower)、無効文字 → `_`、
数字始まりは `_` プレフィックス、予約語は `_` サフィックス(`match` → `match_`)、衝突は
`_2`, `_3`, ... で bump(元名の**ソート順**で早い者勝ち)。**トップレベル tool 名**は加えて
`{connect, credentials, mcp, json, record, prelude}` を避ける(`connect` provider agent /
`credentials` request の再宣言、および default-import qualifier の shadow を防ぐ)。**パラメータ名**は
`{credentials, arguments, arguments_i, mcp, json, record, prelude}` を避ける(`credentials` は body が
`auth = credentials()` で読む request を shadow しないため必須。`url` / `auth` はもう body の値位置に
現れないので seed 不要)。**元の名前**は常に `tool = "..."` 引数と `record.set` の key に乗るので、
サーバーには宣言どおりの名前が届く。出力 synonym は `<mangled>_output`。

tool はトップレベル宣言(block mode)なので、doc annotation は `@"..."` を `agent` の**独立行**に置ける
(旧 `with_tools` 内のローカル agent は line mode ゆえ同一行必須だった)。説明内の改行は `\n`
エスケープで 1 行に畳む。
