## Syntx.hs

### Module data 型の説明。

以下の定義に置いて、modName と modFile は Syntactic には存在しない (どちらも cli によって解決される。) Syntax モジュールにコレを置くことは許容するが、コメントで構文要素ではないことは明示すべき。

```
data Module = Module
  { modFile :: FilePath,
    modName :: Text, -- e.g. "lib.cron" (dot-separated qualified name)
    modDecls :: [Decl]
  }
  deriving (Show)
```

### type 定義とか request 定義などの、param が抽象化されていない。

```
-- request name(params) -> RetType
data RequestDecl = RequestDecl
  { reqAnnot :: Maybe Text,
    reqName :: Text,
    reqParams :: [(Text, Type, Maybe Text)], -- (name, type, annotation)
    reqRet :: Type
  }
  deriving (Show)
```

tuple がコンテキスト外に飛んで行った時、name, type, annotation という関係がわからなくなる。コレは data として切り出すべき。(type 定義や external task 定義などのp共通したパターンにも同じものが使えそう。)

### Effect 定義

### 機能変更 (No-return): return / for break / for next with / finally の廃止

すべての return 文 / for の break / for の next with / for の finally は request システムの break で再現可能なので、廃止します。コレらの control operator は block の分解を複雑にします。また、この変更により祖結合性を保てます。for 文は単にループするだけ。finally も handle の return clause で対応可能です。

これに伴って、IR と Lower の大幅な簡略化も可能だと思います。以下、この機能変更は No-return としてメンションします。
